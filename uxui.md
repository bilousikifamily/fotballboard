# UX/UI Контейнери (Matches UI)

Нижче — список контейнерів у порядку, як вони з’являються на екрані зверху вниз, з вкладеністю та відступами.

## 1) `.panel.profile`
- Роль: верхній блок профілю (логотипи/нікнейм/місце/бали).
- Вміст:
  - логотипи/аватар
  - `h1` (нікнейм)
  - `.stats` (місце/бали)
- Висота:
  - `auto` (по контенту, без фіксованої висоти/`vh`).
- Відступи:
  - зовнішні визначаються контейнером сторінки `.layout` та загальними правилами `.panel`.

## 2) `.panel.matches`
- Роль: секція матчів.
- Вміст:
  - `.section-header` → `.date-switcher`
  - `.matches-list` (горизонтальний скрол)
- Висота:
  - `auto` (по контенту, без фіксованої висоти/`vh`).

### 2.1) `.section-header` → `.date-switcher`
- Роль: перемикач дати (попередня/поточна/наступна).
- Вміст:
  - кнопка назад
  - pill з датою
  - кнопка вперед
- Висота:
  - `auto` (по контенту).
- Відступи:
  - задаються секцією `.panel.matches` (внутрішні gap).

### 2.2) `.matches-list` (горизонтальний scroll)
- Роль: карусель матчів.
- Вміст:
  - `.match-item` (кожна картка)
  - `::before`/`::after` — буфери для центрування першої/останньої картки.
- Висота:
  - `auto` (під висоту `.match-item`), плюс `padding-top`/`padding-bottom` (4px + 18px).
- Відступи:
  - `padding: 4px 0 18px;` (4px зверху від `.date-switcher`, 18px знизу для glow).
  - `gap: 14px` між `.match-item`.
- Логіка:
  - `scroll-snap` + буфери центрують кожну картку по середині.

## 3) `.match-item`
- Роль: контейнер однієї карточки в каруселі.
- Вміст (завжди в цьому порядку):
  1) `.match-time` (час/місто/погода)
  2) `.match` (основна картка матчу)
- Висота:
  - `auto` = висота `.match-time` + `gap` + `.match`.
- Відступи:
  - `gap: 10px` між `.match-time` та `.match`.
- Вирівнювання:
  - `justify-items: center` (центрує контент по ширині).

### 3.1) `.match-time`
- Роль: мета-інформація (час/місто/температура/опади).
- Вміст:
  - `.match-time-row` (рядок 1)
    - `kyiv time` · `CITY` (капслоком) `(local time)` · `temp`
  - `.match-weather-row` (рядок 2)
    - іконка опадів, шкала, відсоток
- Висота:
  - `height: 56px` (фіксована висота контейнера).
- Відступи:
  - `height: 56px` (фіксована висота)
  - `gap: 3px` між рядками
  - `padding-top: 4px` (додатковий відступ зверху)

## 4) `.match` (основна картка)
- Роль: головний блок матчу.
- Вміст (типовий порядок):
  1) `.match-competition` (назва турніру + стадія)
  2) `.match-odds-values` (1 / X / 2)
  3) `.match-header` (логотипи / рахунок / назви)
  4) `.match-average` (середній прогноз)
  5) `.status-closed` (якщо прогнози закриті)
  6) `.prediction-form` (якщо можна прогнозувати)
  7) `.predictions` (список прогнозів)
- Висота:
  - `auto` (по контенту).
- Відступи:
  - `padding: 16px 18px`
  - `gap: 10px`
  - `overflow: visible` (щоб glow кнопки не обрізався)

### 4.1) `.match-competition`
- Роль: назва турніру та стадія (над odds).
- Вміст:
  - `.match-competition-name` (назва турніру)
  - `.match-competition-stage` (стадія)
- Висота:
  - `auto` (по контенту).
- Відступи/позиціювання:
  - `margin: -16px -18px 0` → прилипає до верхніх країв `.match`.
  - `padding: 10px 12px 8px`
  - якщо немає odds → `.is-solo` (округлення знизу).

### 4.2) `.match-odds-values`
- Роль: три контейнери для 1 / X / 2.
- Висота:
  - `auto` (по контенту).
- Відступи/позиціювання:
  - `margin: -16px -18px 0` → прилипає до верхніх країв `.match`.
  - `gap: 0`
  - `border-radius: 18px 18px 12px 12px`
- Підсвітка:
  - `.match-odds-value.is-highlighted` (якщо це обраний/поточний прогноз).

### 4.3) `.match-header`
- Роль: центральний блок з логотипами / рахунком.
- Вміст:
  - логотипи команд
  - рахунок (якщо матч завершений)
- Висота:
  - `auto` (по контенту).

### 4.4) `.match-average`
- Роль: середній прогноз (якщо є дані).
- Висота:
  - `auto` (по контенту).

### 4.5) `.prediction-form`
- Роль: форма прогнозу.
- Вміст:
  - `.score-row` (контроли рахунку)
  - `.match-odds-score` (ймовірність рахунку)
  - `p[data-prediction-status]` (статус)
  - `.prediction-submit` (кнопка ПРОГНОЗ)
- Висота:
  - `auto` (по контенту).
- Відступи:
  - `gap: 8px`

### 4.6) `.prediction-submit`
- Роль: кнопка підтвердження прогнозу.
- Висота:
  - визначається `padding` кнопки (залежить від контенту).
- Відступи/позиціювання:
  - `width: calc(100% + 36px)` + `margin: 0 -18px -16px`
  - прилипає до нижнього краю `.match`
  - радіус: `0 0 18px 18px`

### 4.7) `.predictions`
- Роль: список прогнозів інших гравців.
- Вміст:
  - рядки з нікнеймами та рахунками.
- Висота:
  - `auto` (по кількості рядків).

## Загальна логіка відступів
- Рівень сторінки: більші відступи (18px) між панелями.
- Рівень секції: середні відступи (14px) між елементами.
- Рівень картки: щільні відступи (10px) всередині `.match`.
- Рівень мета-блоку: дрібні відступи (3px) між рядками `.match-time`.

Логіка: чим ближче елементи за змістом — тим менший відступ.
